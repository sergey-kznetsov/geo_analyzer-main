# Загрузка POI через полный официальный рубрикатор 2GIS

Анализатор больше не должен угадывать инфраструктурные рубрики из `config.yaml`.
Основной режим работает так:

1. Геокодер 2GIS определяет координаты и `region_id` точки анализа.
2. Preflight получает полный официальный `rubric/list` этого региона.
3. Загрузчик POI обходит все пригодные `rubric_id` из регионального каталога.
4. Для найденных объектов дозагружается карточка по `id`.
5. Только после этого объекты идут в классификацию, изохроны, карту, отчёт и формулы.

`config.yaml -> dgis.place_queries` теперь не является ограничителем сбора POI.
Он остаётся для совместимости и ручного fallback-режима.

## Основной режим

По умолчанию включён режим полного каталога:

```powershell
python main.py --address "Ижевск, Колизей, жилой комплекс" --refresh-cache
```

Режим также можно задать явно:

```powershell
$env:GEO_ANALYZER_DGIS_PLACE_RUBRIC_SCOPE="all"
$env:GEO_ANALYZER_DGIS_PREFLIGHT_SCOPE="all"
python main.py --address "Пермь, Ленина 50" --refresh-cache
```

## Ручной старый режим

Если нужно временно вернуться к старому ограниченному списку из `config.yaml`:

```powershell
$env:GEO_ANALYZER_DGIS_PLACE_RUBRIC_SCOPE="configured"
$env:GEO_ANALYZER_DGIS_PREFLIGHT_SCOPE="configured"
python main.py --address "Пермь, Ленина 50" --refresh-cache
```

## Каталог 2GIS

Обычный `--refresh-cache` не обновляет официальный каталог 2GIS. Каталог обновляется только явно:

```powershell
python main.py --address "Пермь, Ленина 50" --refresh-cache --refresh-dgis-catalog
```

В GUI для этого используется отдельная галочка `Обновить каталог 2GIS`.

## Важное ограничение API

2GIS Places API не предоставляет один универсальный запрос «вернуть вообще все объекты рядом» без фильтра.
По документации поиск выполняется через параметры запроса, включая `rubric_id`, `type`, `q`, `point`, `radius` и другие. Поэтому корректный способ собрать всё вокруг точки без угадывания рубрик — получить официальный `rubric/list` региона и последовательно опросить все `rubric_id`.

Дополнительно анализатор отдельно запрашивает технические `type`: `station`, `station_platform`, `parking`, `building`.

## Debug-выгрузка

Технические структуры ответа 2GIS сохраняются в:

```text
data/logs/dgis_api_profiles/
```

Excel не засоряется техническими колонками: raw-поля, структуры атрибутов и новые поля API остаются в debug-документах.
