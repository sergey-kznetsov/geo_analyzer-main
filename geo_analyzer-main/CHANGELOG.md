# Changelog

## feature/windows-app-stable-pipeline

### Добавлено

- Простой GUI на Tkinter поверх существующего pipeline: ввод адреса, запуск анализа, блокировка повторного запуска, отображение ошибки, открытие папки результата, открытие Excel-отчёта, создание ZIP.
- CLI-флаг `--refresh-city-benchmark` для принудительного обновления городского бенчмарка.
- Runtime-пути через переменные окружения: `GEO_ANALYZER_OUTPUT_DIR`, `GEO_ANALYZER_CACHE_DIR`, `GEO_ANALYZER_BENCHMARK_DIR`.
- Локальное хранение городского бенчмарка по папкам `data/benchmarks/<city_slug>/`.
- Метаданные бенчмарка: город, дата сборки, источник, параметры, hash конфига, версия схемы.
- Wrapper `pipeline/stable_runner.py`, который не переписывает основной pipeline, а добавляет app-level метаданные и явные пути результата.

### Изменено

- CLI теперь запускает stable runner, сохраняя прежний пользовательский формат запуска.
- GUI запускает тот же stable runner, что и CLI.
- `places_loader.py` обновлён до версии кеша `places_loader_v5_rubric_merge` и объединяет POI, пришедшие из нескольких рубрик 2GIS.
- README обновлён: запуск CLI, GUI, `.env`, структура данных, городской бенчмарк, сборка Windows exe.

### Зафиксированные спорные решения

- Городской бенчмарк сейчас работает как локальный baseline первого успешного анализа города. Это сделано без агрессивного изменения существующих метрик. Полноценный city-grid benchmark нужно добавлять отдельным сборщиком контрольных точек города.
- Старые OSM/Yandex-модули не удалены, потому что без полного локального прогона импортов безопаснее оставить их как legacy-код.
- GitHub Actions workflow и build scripts не добавлены через текущий инструмент: создание workflow/build-файлов блокировалось системой безопасности коннектора. Сборка PyInstaller описана в README.

### Требует проверки локально

- `python main.py --address "Ижевск, Пушкинская 277"`
- `python main.py --lat 56.8526 --lon 53.2115`
- `python gui.py`
- создание `report.xlsx`, `summary.txt`, `meta.json`
- создание ZIP из GUI
- сборка `GeoAnalyzer.exe` через PyInstaller
